# Проблема
(Есть еще нюансы, но для начала нужно определиться с это архитектурой)

*Единственный уникальный параметр это ***id*** сущности*
*Но что бы не заставлять пользователя ковырятся на платформе в поисках этого ***id*** можно делать поиск по именни*
*Но тогда может вернуться несколько сущностей с одинаковым именем*
### Предложение
*Использовать функцию поиска как первоначальную точку входа со специальными параметрами что бы при получении вытягивать ***id*** и какой-нибудь отличительный параметр что бы модель могла задать уточняющий вопрос и затем вызывать нужную функцию*
### Например
~~~text
"""
Создай на доске Работа лист с именем Обязанности
"""
~~~
*досок вернулось несколько*
~~~text
"""
Было найдено n досок с именем 'Работа'
'Работа' с листом 'Сотрудники'
'Работа' с листом 'Фриланс'
"""
~~~
*Где пользователь отвечает конкретно и уже вытгивается ***id*** нужной доски для манипуляций*

# Trello Search

## Модуль
Функция `trello_search` работает с **Trello Search API** (`/search`).  
Авторизация через переменные окружения: `API_KEY_TRELLO`, `API_TOKEN_TRELLO`.

## Возможности
- Поиск по доскам (`boards`)
- Поиск по карточкам (`cards`)
- Поиск по участникам (`members`)
- Поиск по организациям (`organizations`)

## Возвращаемое значение
JSON‑строка с нормализованным результатом поиска или ошибкой.

# Trello Label Action

## Модуль
Функция `trello_label_action` работает с **Trello API** (`/labels`).  
Авторизация через переменные окружения: `API_KEY_TRELLO`, `API_TOKEN_TRELLO`.

## Возможности
- **create** — создать метку  
- **get** — получить метку по ID  
- **update** — обновить имя или цвет метки  
- **delete** — удалить метку  

## Возвращаемое значение
JSON‑строка с результатом операции или ошибкой.
